# 04 Test And Fix Loop

默认最多 3 轮，除非 `run.yaml` 覆盖 `execution.max_fix_iterations`。

## 每轮流程

1. 在远端隔离 user 下执行 YAML 中的测试命令。
2. 保存完整 stdout/stderr 到 `logs/`。
3. 判断失败是否来自环境、配置、依赖、测试数据、真实 bug。
4. 如果是环境或凭据缺失，明确阻塞并索要缺失项。
5. 如果是业务 bug，在本地业务仓库修改代码。
6. 重新同步本地仓库到远端。
7. 重新运行最小复现测试，再运行最终真实集成门禁。

## 证据要求

报告中的结论必须有命令、日志路径和关键摘录支撑。

通过结论至少需要：

- 最终真实集成测试命令。
- 退出码。
- 关键通过输出。
- 外部测试命名空间或 smoke token。

失败结论至少需要：

- 当前根因。
- 已尝试的修复。
- 仍缺少的日志、权限、凭据、复现步骤或文件。

## 修复边界

- 只改本地业务仓库，不在远端副本直接修复。
- 不自动 commit/push。
- 不改用户明确禁止的目录或文件。
- 不把 mock 通过包装成真实集成通过。
