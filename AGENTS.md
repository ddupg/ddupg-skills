# AGENTS.md

## 项目定位

这里存放自用的通用 skills。每个 skill 应该是一个独立目录，核心说明放在该目录的 `SKILL.md` 中。

## 维护规则

- 新增、删除或重命名 skill 后，同步更新根目录 `README.md` 中的 skills 表格。
- `SKILL.md` 应使用标准 skill 格式，避免绑定单一 agent 的私有能力，尽量兼容主流 agent 的读取和执行方式。
- 修改已有 skill 时，优先保持原有触发条件、工作流和输出格式稳定。
- 不要引入和当前 skill 无关的配置、依赖、脚本或目录结构。
- 辅助脚本、模板、示例文件只在对应 skill 确实需要时添加，并放在该 skill 目录内。
- 文档默认使用中文；代码标识符、命令、API 名称可以保留英文。
